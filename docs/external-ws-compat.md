# 外部 WebSocket 协议兼容说明

本文档用于补充与 `FUNASR_EXTERNAL_API_DOC.md` 对齐的 WebSocket 协议说明，不影响现有 HTTP 接口（`/healthz`、`/asr`）的使用。

## 连接地址

- 容器内服务监听端口仍为 `8000`。
- 建议在 Compose 同时映射：
  - `8000:8000`
  - `10095:8000`（兼容旧客户端默认端口）
- 可用连接地址（root path 与 `/ws` 均可用）：
  - `ws://<host>:10095/`（推荐）
  - `ws://<host>:10095/ws`
  - `ws://<host>:8000/`
  - `ws://<host>:8000/ws`

## WebSocket 子协议兼容

- 若客户端在握手中请求 `Sec-WebSocket-Protocol: binary`，服务端会回选 `binary`。

## 消息帧顺序

客户端发送顺序固定为三段：

1. **首帧 JSON**：会话开始与音频元信息。
2. **binary 音频帧**：音频原始字节流，可分多帧发送。
3. **结束 JSON**：会话结束标记（表示音频发送完毕）。

## 首帧 JSON 字段（宽松兼容）

以下字段按“尽量兼容旧客户端”处理，允许缺省或等价格式：

- `wav_name`：
  - 含义：音频名（仅用于标识与回传）。
  - 兼容：字符串；缺省时可由服务端使用默认名。
- `wav_format`：
  - 含义：音频格式（如 `pcm`、`wav`）。
  - 兼容：大小写不敏感；缺省时按服务端默认处理。
- `audio_fs`：
  - 含义：采样率（Hz）。
  - 兼容：数字或数字字符串（如 `16000` / `"16000"`）；缺省时按服务端默认处理。
- `hotwords`：
  - 含义：热词提示。
  - 兼容：可为字符串、字符串数组或对象（旧客户端格式差异较大时按宽松解析思路处理）；缺省可不传。

## binary 音频帧

- 使用二进制帧发送音频数据。
- 支持分片连续发送，直到结束帧 JSON 发出。

## 结束 JSON

- 用于显式标记“音频发送结束”。
- 建议至少包含结束语义字段（例如 `is_speaking: false` 或等价结束标记），以兼容外部客户端习惯。

## 返回字段（兼容约定）

服务端返回结果中，兼容关注以下字段：

- `text`：识别文本
- `mode`：识别模式/阶段标识
- `wav_name`：对应输入音频名
